2. 目录也是架构的一部分
- src 源代码
- bin 可执行文件(打包给玩家)
- build 编译过程产物
- include 放.h文件
- lib 放链接库.dll或.a
- assets 资源目录: 图片、文本、模型、Shader......
- doc 文档
- 根目录放一些编译脚本

- raylib.com 可查看官方文档

4. 什么是架构？
- 建筑框架设计：供水、供强电、供弱电、楼层、房间
- 程序构架：实体(类型)、模块、控制流(调用关系)、依赖流(类型与类型/函数与函数)
  - 生成: 使用工厂Factory
  - 存储: 使用上下文Context + 仓库Repository
  - 控制: 使用Business, Controller
    - 输入 -> 逻辑 -> 画
  - 行为: 使用领域Domain, 生成/移除/移动
- 模块
  - 输入模块
  - 资源管理模块
- 控制流
  - main -> B_ -> Do_ -> Context -> Repo_ -> E_
- 架构的作用
  - 协作：有人负责main，有人负责Do_，有人负责B_，三方互不影响

5. 什么是实体：
  - 程序最重要的部分
  - 角色、植物、环境、车辆
  - 账号、一条动态、一条评论

如果编译报错, 请创建 build 和 bin 目录
